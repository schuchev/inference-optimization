# Отчёт по оптимизации инференса rubert-mini-frida

В ходе домашнего задания были реализованы три подхода:

1. **Part 1 — Базовый**  
   - Простая модель на CPU, без оптимизаций.  
   - Получили высокое latency (~168 ms) и низкий throughput (~289 RPS).

2. **Part 2 — ONNX**  
   - Модель сконвертирована в ONNX, инференс через ONNX Runtime.  
   - Latency уменьшился почти в 2.5 раза, throughput вырос до ~709 RPS.  
   - Значительное ускорение даже на CPU.

3. **Part 3 — ONNX + Dynamic Batching**  
   - Реализовано динамическое батчирование для обработки нескольких запросов одновременно.  
   - Latency_mean снизился до 43.6 ms, throughput увеличился до ~1072 RPS.  
   - Эффективно при высокой параллельной нагрузке; для маленьких моделей на последовательных запросах прирост минимален.

**Выводы:**

- Конвертация в ONNX даёт самый большой прирост производительности.  
- Динамическое батчирование повышает throughput и снижает средний latency только при высокой параллельной нагрузке.  
- Для маленьких моделей на CPU эффект динамического батчинга зависит от нагрузки: при последовательных запросах улучшение минимально, но при высокой параллельной нагрузке batching заметно увеличивает throughput и снижает средний latency. На GPU и для больших моделей batching даёт ещё более значительное ускорение.
